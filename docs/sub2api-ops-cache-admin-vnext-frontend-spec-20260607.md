## 2. 前端实现原则

1. 页面面向后台管理和运维人员，信息密度高但层级清晰。
2. 运维监控优先回答：现在是否出事、哪里出事、影响谁、下一步做什么。
3. 缓存管理优先回答：开关是否生效、命中率如何、为什么没命中、如何清理。
4. 所有配置页必须具备默认值、字段校验、保存反馈、失败提示。
5. 所有异步操作必须有加载中、成功、失败、重复提交保护。
6. 所有敏感字段必须按 PRD 第 18 章脱敏展示。
7. 所有列表必须具备空态、加载失败态、分页和默认排序。

### 12.1 高级统计展示区

高级缓存策略页面在配置表单下方固定展示四个区域，数据来自高级缓存统计接口。高级缓存总开关关闭时四个区域继续展示历史和当前精确缓存相关统计，并在页首显示“高级策略未启用，当前线上请求使用精确缓存”。高级缓存异常时显示“已回退到精确缓存”，主统计区不隐藏。

#### 容量控制区

| 字段 | 展示 | 空态/失败态 | 权限 |

| 价格缺失模型 | 标签组 | 无缺失显示“价格配置完整” | 平台所有者、运营 |

权限统一规则：平台所有者可编辑高级缓存配置并查看全部高级统计；运维、运营可进入页面查看高级统计，但高级配置表单只读且保存按钮隐藏；客服不可见高级缓存策略页面。无权限直达时提示“当前账号无权限执行此操作”。所有统计加载失败时保留上次成功数据并显示失败提示；没有历史成功数据时显示 `--` 和空态文案。

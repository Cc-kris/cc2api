## 2. 验收目标

1. 运维监控减少少量错误导致的无效告警。
2. 运维人员能通过统一错误中心快速判断错误来源、影响范围和处理方向。
3. AI 运维分析支持自动与手动场景，并输出可用结论。
4. 缓存管理完成独立菜单、配置、统计、清理、多平台缓存和高级能力。
5. 用户管理、账号管理、后台仪表盘按 PRD 完成调整。
6. 权限、脱敏、异常降级、灰度、回滚规则符合上线要求。
7. 阶段 1-3 达到主版本上线标准；阶段 4-5 达到灰度上线标准。

## 4. 分阶段验收

### 4.1 阶段 1：运维与后台基础

| 验收项 | 验收标准 | 通过条件 |
|---|---|---|
| 运维告警重构 | 1 分钟窗口结合最终失败数、失败率、样本量、影响范围判断 | 1-2 条最终失败不会触发 P0/P1 强告警 |
| 统一错误中心 | 原上游错误、客户端错误、请求错误进入统一错误中心 | 旧入口自动带入分类筛选 |
| 统一错误详情 | 展示请求链路、错误分类、影响范围、原始记录 | 运维能判断问题来源 |
| AI 分析基础配置 | AI 未配置置灰，配置后可测试连接 | 成功/失败提示明确 |
| 缓存管理菜单 | 系统设置新增“缓存管理” | 原通用设置缓存区迁移成功 |
| 用户余额筛选 | 用户管理列表支持余额筛选 | 可筛低余额、高余额、余额区间用户 |
| 账号编辑调整 | 编辑账号页不再出现上游预警金额和邮件通知字段 | 保存账号不校验已移除字段 |
| 仪表盘充值与入账 | 新增入账概览和复购分布 | 排除超管账号，人工加款计入有效入账 |

阶段 1 通过标准：P0 功能全部通过；旧入口无死链；异常不影响主请求链路。

### 4.2 阶段 2：OpenAI 缓存产品化

| 验收项 | 验收标准 | 通过条件 |
|---|---|---|
| OpenAI 精确缓存 | 相同 API Key、分组、平台、模型、请求体可命中 | 重复请求命中并返回一致响应 |
| 缓存配置 | 全局开关、平台开关、TTL、大小、模型名单生效 | 保存后新请求按最新配置执行 |
| 缓存统计 | 展示输入、输出、命中 tokens、请求命中率、tokens 命中率 | 口径与 PRD 一致 |
| 缓存清理 | 按平台、模型、API Key、分组、时间范围清理 | 清理后对应缓存不命中 |
| 原因分析 | 展示绕过原因和写入跳过原因 | 不命中原因可追踪 |

阶段 2 通过标准：OpenAI 重复请求请求命中率和 tokens 命中率达到 PRD 目标；缓存失败不影响主请求。

### 4.3 阶段 3：Claude/Gemini 精确缓存

| 验收项 | 验收标准 | 通过条件 |
|---|---|---|
| Claude 缓存 | Claude 文本类 HTTP/SSE 请求支持精确缓存 | 完整响应后写入，重复请求命中 |
| Gemini 缓存 | Gemini 文本类 HTTP/SSE 请求支持精确缓存 | 流式完整结束后写入 |
| 平台隔离 | 三个平台缓存互不串用 | 同内容不同平台不会互相命中 |
| 绕过规则 | 工具调用、图片、文件、非确定性请求按规则绕过 | 绕过原因正确记录 |
| 统计汇总 | 可按平台、模型筛选统计 | 三个平台数据可分别查看 |

阶段 3 通过标准：三平台精确缓存稳定，不跨用户、API Key、分组、平台、模型误命中。

### 4.4 阶段 4：高级缓存策略

| 验收项 | 验收标准 | 通过条件 |
|---|---|---|
| 容量上限 | Redis 容量上限配置可保存并生效 | 超容量按淘汰策略处理 |
| 压缩存储 | 达到阈值响应按规则压缩 | 读取后响应正确 |
| 淘汰策略 | 支持 LRU/LFU/W-TinyLFU | 切换时有影响提示 |
| 热点分析 | 展示热点模型、API Key、缓存 Key 摘要 | 可用于判断缓存收益 |
| 成本节省 | 本地缓存节省和上游 Prompt Cache 节省分开展示 | 价格缺失展示空态 |

阶段 4 通过标准：默认关闭，灰度开启，异常回退精确缓存。

### 4.5 阶段 5：语义相似缓存

| 验收项 | 验收标准 | 通过条件 |
|---|---|---|
| 默认关闭 | 未开启时不产生语义命中 | 默认关闭 |
| 语义模型配置 | 服务地址、API Key、模型、维度、规则版本可配置 | 测试连接成功后才允许启用 |
| 观察阶段 | 只记录候选，不返回缓存 | 可查看审计 |
| 人工审核阶段 | 人工确认后才允许命中 | 审核记录完整 |
| 灰度阶段 | 仅灰度 API Key 生效 | 非灰度 Key 不命中 |
| 正式阶段 | 质量验收后正式启用 | 误命中率、投诉率低于阈值 |
| 自动回滚 | 超阈值自动关闭 | 页面展示原因 |

阶段 5 通过标准：隔离范围正确，异常自动绕过，未完成观察和审核不得正式启用。

## 8. 异常降级验收

| 异常场景 | 验收标准 |
|---|---|
| AI 未配置 | AI 按钮置灰，不影响错误中心 |
| AI 连接失败 | 测试连接失败提示明确 |
| AI 超时 | 任务失败，可重试 |
| AI 队列满 | 提示稍后重试，不重复创建 |

| 缓存读取失败 | 自动请求上游 |
| 缓存写入失败 | 返回上游结果，记录原因 |
| 缓存清理失败 | 展示失败原因，不误删其他缓存 |
| 高级缓存异常 | 回退精确缓存 |
| 语义模型异常 | 语义缓存绕过，主请求继续 |
| 语义误命中超阈值 | 自动关闭并展示原因 |
| 统计服务失败 | 展示失败和重试入口 |
| 旧入口访问 | 跳转新入口，不出现 404 |

## 9. 上线前阻断条件

存在以下任一情况，不允许上线：

1. 主请求因 AI、缓存、统计异常被阻塞。
2. 缓存出现跨用户、跨 API Key、跨分组、跨模型误命中。
3. 语义缓存默认开启。
4. 敏感字段在页面、复制、导出或 AI 上下文中明文泄露。
5. 旧缓存配置或告警规则迁移后丢失。
6. 账号编辑页仍出现上游预警金额或相关邮件通知。
7. 小样本错误仍触发 P0/P1 强告警。
8. 充值与入账统计未排除超管账号。
9. 人工加款未计入有效入账。
10. 未支付或取消订单被计入有效入账。

## 12. 最终验收判定规则

1. 阶段 1-3 全部通过后，本 PRD 主版本判定为通过。
2. 阶段 4-5 可独立灰度验收，不阻塞阶段 1-3 主版本上线。
3. 任一 P0 功能未通过，整体结论为不通过。
4. 任一敏感信息泄露问题未修复，整体结论为不通过。
5. 任一主请求阻塞问题未修复，整体结论为不通过。
6. 任一跨用户、跨 API Key、跨分组、跨模型缓存误命中问题未修复，整体结论为不通过。
7. 阶段 5 语义缓存未完成观察和人工审核，不允许正式启用。
8. 所有遗留问题必须记录责任人、修复时间和复验结论。

Codex APP 配置说明

一、这一节适合谁

如果你使用的是桌面版 Codex APP，并且希望通过 cc-ai 使用 gpt-5.5，就按这一节操作。

重点：Codex APP 只需要修改 config.toml。不要去改认证文件，也不要再额外添加环境变量。
系统配置文件路径要做什么
macOS / Linux~/.codex/config.toml创建或打开这个文件，把下方内容写进去。
Windows%userprofile%\.codex\config.toml创建或打开这个文件，把下方内容写进去。

二、先准备 API Key

  1. 登录 https://cc-ai.xyz。
  2. 进入 API Key 页面。
  3. 创建一个新的 API Key。
  4. 复制完整 Key，后面要替换示例里的 填写API秘钥。

三、找到配置目录

macOS / Linux

  1. 打开终端。
  2. 执行 mkdir -p ~/.codex，确保目录存在。
  3. 用编辑器打开 ~/.codex/config.toml。如果文件不存在，就新建一个。

Windows

  1. 按 Win + R。
  2. 输入 %userprofile%\.codex 并回车。
  3. 如果没有 .codex 文件夹，就在用户目录下新建。
  4. 在里面新建或打开 config.toml。

四、改之前先备份

如果你之前已经能正常使用 Codex APP，先做一个备份，这样改错时可以快速恢复。

  1. 在同一个目录里找到 config.toml。
  2. 复制一份，命名为 config.toml.bak。
  3. 确认备份文件和原文件在同一个目录。
  4. 再开始修改原来的 config.toml。
不要做的事原因
不要只改备份文件Codex APP 读取的是 config.toml，不是 config.toml.bak。
不要把文件命名成 config.toml.txtWindows 可能隐藏扩展名，新手容易误建成文本文件。
不要把示例里的 API Key 占位文字留着必须换成你自己的真实 API Key 才能通过验证。

五、写入 config.toml

复制下面整段内容，替换原来的 Codex 配置。然后把 填写API秘钥 换成你的真实 API Key。

model_reasoning_effort = "high"

approval_policy = "never"
model = "gpt-5.5"
model_auto_compact_token_limit = 945000
model_context_window = 1050000
model_provider = "ccai"
personality = "pragmatic"
sandbox_mode = "danger-full-access"
tool_output_token_limit = 6000

[model_providers.ccai]
base_url = "https://cc-ai.xyz"
name = "OpenAI"
experimental_bearer_token = "填写API秘钥"
wire_api = "responses"
requires_openai_auth = false
supports_websockets = true

[features]
goals = true
multi_agent = true
image_generation = true

child_agents_md = true
hooks = true
suppress_unstable_features_warning = true
memories = true

六、保存并重启

  1. 保存 config.toml。
  2. 完全退出 Codex APP。
  3. 重新打开 Codex APP。
  4. 发起一次简单对话，确认可以正常返回。

七、检查清单

检查项结果
config.toml 已保存到正确目录是
experimental_bearer_token 已替换成真实 API Key是
model 是 gpt-5.5是
model_provider 是 ccai是
base_url 是 https://cc-ai.xyz是
Codex APP 已完全重启是

八、常见问题

1. 文件夹不存在怎么办？

直接新建即可。Codex 不要求这个目录提前存在。

2. 提示鉴权失败怎么办？

先检查 experimental_bearer_token 是否还是示例文字，或者复制 Key 时是否漏了一段。

3. 需要再配置其他密钥文件吗？

不需要。这个版本的说明只要求写 config.toml。

Codex CLI 配置说明

一、这一节适合谁

如果你使用的是命令行里的 Codex CLI，就按这一节操作。CLI 和桌面版使用同一套 .codex/config.toml 配置。

重点：Codex CLI 也只需要修改 config.toml。不要把 API Key 写到其他认证文件里。

二、确认路径

系统路径说明
macOS / Linux~/.codex/config.toml终端默认会读取这个用户目录下的配置。
Windows%userprofile%\.codex\config.toml在当前 Windows 用户目录下创建或修改。

三、先确认你正在改当前用户的配置

Codex CLI 读取的是当前系统用户目录下的配置。如果你用管理员窗口、远程服务器或不同系统用户运行 CLI，要确认配置写在运行 CLI 的那个用户目录里。

场景检查方法
macOS / Linux在终端执行 echo $HOME，确认这里就是你要配置的用户目录。
Windows在命令行执行 echo %userprofile%，确认显示的是当前用户目录。
远程服务器在远程服务器里创建配置，不是在本机电脑里创建。

四、写入配置

复制下方内容到 config.toml。如果你已有其他 Codex 配置，先备份旧文件，再整体替换，避免新手合并时漏字段。

model_reasoning_effort = "high"

approval_policy = "never"
model = "gpt-5.5"
model_auto_compact_token_limit = 945000
model_context_window = 1050000
model_provider = "ccai"
personality = "pragmatic"
sandbox_mode = "danger-full-access"
tool_output_token_limit = 6000

[model_providers.ccai]
base_url = "https://cc-ai.xyz"
name = "OpenAI"
experimental_bearer_token = "填写API秘钥"
wire_api = "responses"
requires_openai_auth = false
supports_websockets = true

[features]
goals = true
multi_agent = true
image_generation = true

child_agents_md = true
hooks = true
suppress_unstable_features_warning = true
memories = true

五、命令行验证

  1. 保存文件后关闭当前终端窗口。
  2. 重新打开一个新终端。
  3. 运行 Codex CLI。
  4. 输入一个简单任务，例如“用一句话介绍当前模型”。

六、排错顺序

现象先检查什么
提示 Key 无效检查 experimental_bearer_token 是否是完整 API Key。
提示模型不可用检查 model = "gpt-5.5" 是否写错。
提示连接失败检查 base_url = "https://cc-ai.xyz" 是否多写了路径或斜杠。
修改后没有变化确认改的是当前用户目录下的 config.toml，并且已重新打开终端。

七、新手最容易改错的地方

  • 把配置写到了项目目录，但 CLI 实际读取的是用户目录。
  • 复制配置时漏掉了 [model_providers.ccai] 这一行。
  • 把 base_url 写成了带 /v1 的地址。Codex 这里按示例保持 https://cc-ai.xyz。
  • 终端没有重新打开，旧进程还在使用旧配置。

Claude Code 配置说明

一、这一节适合谁

如果你使用 Claude Code，需要按 Claude Code 自己的配置方式来写。它不是 Codex 客户端，所以不要使用 Codex 的配置目录。

重点：Claude Code 使用 ~/.claude/settings.json 里的 env 配置。

二、配置文件路径

系统路径说明
macOS / Linux~/.claude/settings.json没有文件就新建。
Windows%userprofile%\.claude\settings.json没有文件就新建。

三、改之前先备份

如果 settings.json 已经存在，先复制一份命名为 settings.json.bak。Claude Code 的配置通常还包含主题、通知、权限等内容，备份可以避免误删。

四、第一次配置时写入

如果你的 settings.json 原来是空的，直接写入下面内容，并把 填写API秘钥 替换成真实 API Key。

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://cc-ai.xyz",
    "ANTHROPIC_AUTH_TOKEN": "填写API秘钥",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1",
    "CLAUDE_CODE_ATTRIBUTION_HEADER": "0"
  }
}

五、已有其他配置时怎么合并

如果文件里已经有主题、通知等设置，不要删除原字段，只需要增加或更新 env。示例：

{
  "theme": "dark",
  "notifications": true,
  "env": {
    "ANTHROPIC_BASE_URL": "https://cc-ai.xyz",
    "ANTHROPIC_AUTH_TOKEN": "填写API秘钥",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1",
    "CLAUDE_CODE_ATTRIBUTION_HEADER": "0"
  }
}

六、保存并重启

  1. 保存 settings.json。
  2. 完全退出 Claude Code。
  3. 重新打开 Claude Code。
  4. 发起一次简单请求验证。

七、检查清单

检查项结果
ANTHROPIC_BASE_URL 是 https://cc-ai.xyz是
ANTHROPIC_AUTH_TOKEN 已替换成真实 API Key是
JSON 使用英文双引号和英文冒号是
Claude Code 已完全重启是

八、常见问题

1. 保存后还是不生效怎么办？

先确认 Claude Code 已完全退出再打开。只关闭窗口但进程还在时，配置可能没有重新加载。

2. JSON 报错怎么办？

检查逗号、双引号和大括号。中文引号会导致配置无法识别。

九、字段解释

字段作用
ANTHROPIC_BASE_URL告诉 Claude Code 请求发到 cc-ai。
ANTHROPIC_AUTH_TOKEN你的 cc-ai API Key。
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC减少非必要外部流量。
CLAUDE_CODE_ATTRIBUTION_HEADER关闭额外标识头，保持请求更干净。

OpenCode 配置说明

一、这一节适合谁

如果你使用 OpenCode，需要按 OpenCode 自己的配置文件来写。它不是 Codex 客户端，所以不要去改 Codex 的配置目录。

重点：OpenCode 只改 opencode.json 或 opencode.jsonc。这里把 API Key 直接写进配置文件，不需要再走客户端里的连接命令。

二、选择全局配置还是项目配置

使用场景文件路径适合谁
所有项目都使用 cc-ai~/.config/opencode/opencode.json新手推荐。
Windows 全局配置%USERPROFILE%\.config\opencode\opencode.jsonWindows 用户。
只让某一个项目使用 cc-ai项目根目录/opencode.json多个项目要用不同 Provider 时使用。

三、全局和项目配置怎么选

新手直接用全局配置。只有当你明确知道“这个项目要用 cc-ai，另一个项目不用 cc-ai”时，再使用项目根目录里的配置。

选择优点风险
全局配置配置一次，所有项目都能用。所有 OpenCode 项目都会默认走 cc-ai。
项目配置只影响当前项目。换项目后需要重新配置。

四、创建配置文件

  1. 先决定使用全局配置还是项目配置。
  2. 如果目录不存在，先手动创建目录。
  3. 新建 opencode.json。
  4. 把下面整段 JSON 写进去。

五、写入 opencode.json

把 填写API秘钥 替换成真实 API Key。baseURL 保持 https://cc-ai.xyz/v1。

{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "ccai": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "cc-ai",
      "options": {
        "baseURL": "https://cc-ai.xyz/v1",
        "apiKey": "填写API秘钥"
      },
      "models": {
        "gpt-5.5": {
          "name": "GPT-5.5",
          "limit": {
            "context": 1050000,
            "output": 128000
          },
          "options": {
            "store": false
          },
          "variants": {
            "low": {},
            "medium": {},
            "high": {},
            "xhigh": {}
          }
        }
      }
    }
  },
  "model": "ccai/gpt-5.5",
  "agent": {
    "build": {
      "options": {
        "store": false
      }
    },
    "plan": {
      "options": {
        "store": false
      }
    }
  }
}

六、保存并重启

  1. 保存 opencode.json。
  2. 完全退出 OpenCode。
  3. 重新打开 OpenCode。
  4. 确认当前模型是 ccai/gpt-5.5。
  5. 发起一次简单请求验证。

七、检查清单

检查项结果
opencode.json 已创建或已修改是
provider 里有 ccai是
baseURL 是 https://cc-ai.xyz/v1是
apiKey 已替换成真实 API Key是
model 是 ccai/gpt-5.5是
OpenCode 已完全重启是

八、常见问题

1. 全局配置和项目配置同时存在时用哪个？

项目根目录里的 opencode.json 优先级更高，会覆盖全局配置。

2. 可以写成 https://cc-ai.xyz 吗？

OpenCode 这里按 OpenAI-compatible 方式接入，推荐写完整的 https://cc-ai.xyz/v1。

3. 需要再执行连接命令吗？

不需要。本说明采用“只改配置文件”的方式，API Key 已在 opencode.json 里填写。

九、OpenCode 常见错误

错误处理方式
JSON 末尾多了逗号删除最后一个字段后面的逗号。
baseURL 少了 /v1按示例改成 https://cc-ai.xyz/v1。
模型写成 gpt-5.5OpenCode 当前示例里要写完整的 ccai/gpt-5.5。
配置了全局文件但项目里还有旧配置项目配置优先级更高，先检查项目根目录是否还有旧 opencode.json。

CC Switch 配置说明

一、这一节适合谁

如果你使用 CC Switch 这类图形界面工具，请按界面里的 Provider 表单填写。它不需要改 Codex 或 OpenCode 的配置文件。

重点：CC Switch 是图形界面配置，按字段填写 Provider、保存、启用即可。

二、新增 Provider

  1. 打开 CC Switch。
  2. 进入 Providers 或 Provider 管理页面。
  3. 点击 New / Add / Custom Provider。
  4. Provider Name 填 cc-ai 或 ccai。

三、填写接口信息

字段名填写内容说明
Provider Namecc-ai方便后续识别。
Base URLhttps://cc-ai.xyz如果界面要求 OpenAI-compatible 地址，可填 https://cc-ai.xyz/v1。
API Key填写API秘钥替换成你创建的真实 API Key。
Default Modelgpt-5.5默认模型。

四、先确认协议类型

不同版本的 CC Switch 字段名称可能不完全一样。看到 OpenAI、OpenAI-compatible、Custom OpenAI 时，Base URL 通常填写带 /v1 的地址；看到 Claude、Anthropic 时，Base URL 通常填写不带 /v1 的地址。

五、保存并启用

  1. 点击 Save 保存 Provider。
  2. 在 Provider 列表中选择刚刚创建的 cc-ai。
  3. 点击 Activate / Enable / Use，确保当前启用的是这个 Provider。
  4. 发起一次简单请求验证。

六、如果界面分 Claude / OpenAI 两种 Provider

如果 CC Switch 让你选择协议类型：

协议类型Base URLKey 字段模型
OpenAI-compatiblehttps://cc-ai.xyz/v1API Keygpt-5.5
Claude / Anthropichttps://cc-ai.xyzANTHROPIC_AUTH_TOKEN选择界面支持的 Claude 模型

七、检查清单

检查项结果
CC Switch 中能看到新 Provider是
Base URL 已按协议类型填写是
API Key 已替换成真实 API Key是
默认模型已设置是
已点击 Save是
已点击 Activate / Enable是

八、常见问题

1. 保存成功但没有走中转怎么办？

先确认当前启用的是刚刚保存的 Provider，再确认 Base URL 是否填在正确的协议类型下。

2. Base URL 到底要不要带 /v1？

OpenAI-compatible 通常带 /v1；Claude / Anthropic 风格通常不带。界面如果有明确示例，以界面要求为准。

3. 可以截图让别人帮忙检查吗？

可以截图字段名和 Base URL，但不要把真实 API Key 截进去。
